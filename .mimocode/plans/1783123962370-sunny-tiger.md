# Dashboard Layout Improvement Plan

## Problem
The admin dashboard main content area is constrained by `max-w-7xl` (1280px max width), which causes:
- Empty space on large/ultrawide monitors
- Content doesn't fill the available space next to the sidebar
- The layout looks cramped and not utilizing screen real estate

## Current State
- **File**: `src/app/admin/layout.tsx` (line 153)
- **Current classes**: `flex-1 p-6 md:p-10 max-w-7xl mx-auto w-full min-h-[calc(100vh-60px)] md:min-h-screen`
- Sidebar width: `w-64` (256px)

## Solution
Remove the `max-w-7xl mx-auto` constraint from the main content area, allowing it to fill all available space next to the sidebar.

### Changes

**File: `src/app/admin/layout.tsx`**

Line 153 - Change:
```tsx
<main className="flex-1 p-6 md:p-10 max-w-7xl mx-auto w-full min-h-[calc(100vh-60px)] md:min-h-screen">
```

To:
```tsx
<main className="flex-1 p-6 md:p-10 min-h-[calc(100vh-60px)] md:min-h-screen">
```

This single change will:
1. Remove the 1280px max-width constraint
2. Remove the `mx-auto` centering (not needed in a flex layout)
3. Remove `w-full` (redundant with `flex-1`)
4. Let the content fill all available space
